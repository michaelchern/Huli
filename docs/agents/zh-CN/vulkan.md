# Huli Vulkan Context

只在 Vulkan 概念、验证层、GPU/驱动问题、shader、同步、descriptor、swapchain 或渲染现象解释任务中加载本文件。

## 学习姿势

- 先解释概念目的，再说明参考来源，最后说明 Huli 当前是否已经实现。
- 用户表示没看懂时，优先用直觉和可视化因果解释，再给 Vulkan API 细节。
- 不要把外部示例的实现约束当作 Huli 的长期架构规则；Huli 可以为了学习做更小的实验。

## 调试规则

- 先找第一条真实错误：`VK_ERROR`、VUID、shader 编译错误、MSVC `error C...`，不要被包装日志带偏。
- 如果涉及另一台机器或另一个仓库的日志，明确证据来源，不要用 Huli 本地状态替代复现。
- 修改 shader / C++ / CMake 后运行最小相关验证；无法运行时说明原因。
